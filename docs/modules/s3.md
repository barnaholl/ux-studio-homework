# S3 Module

## Overview

Thin wrapper around AWS SDK v3 for DigitalOcean Spaces (S3-compatible). Used for avatar storage.

## Files

| File | Role |
|------|------|
| `s3/s3.service.ts` | S3 operations: getObject, upload, delete, listByPrefix, getCdnUrl |
| `s3/s3.module.ts` | @Global() module exporting S3Service |

## Configuration (Environment Variables)

| Variable | Default | Description |
|----------|---------|-------------|
| `DO_SPACES_REGION` | `ams3` | DigitalOcean Spaces region |
| `DO_SPACES_ENDPOINT` | `https://{region}.digitaloceanspaces.com` | Spaces endpoint |
| `DO_SPACES_BUCKET` | `''` | Bucket name |
| `DO_SPACES_KEY` | `''` | Access key ID |
| `DO_SPACES_SECRET` | `''` | Secret access key |

## Methods

| Method | Signature | Description |
|--------|-----------|-------------|
| `getCdnUrl()` | `() => string` | Returns `{endpoint}/{bucket}` |
| `getObject(key)` | `(key: string) => Promise<Buffer>` | Download object as Buffer |
| `upload(key, body, contentType)` | `(string, Buffer, string) => Promise<string>` | Upload; returns CDN URL |
| `delete(key)` | `(key: string) => Promise<void>` | Delete single object |
| `listByPrefix(prefix)` | `(prefix: string) => Promise<string[]>` | List all keys under prefix |

## Notes

- `forcePathStyle: false` — required for virtual-hosted-style URLs on Spaces
- No unit tests: the service is a thin SDK wrapper; behavior is validated in e2e tests against a real or mocked bucket
- In unit tests, S3Service is mocked with `jest.fn()` per method
